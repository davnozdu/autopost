# autopost

Docker-сервис автопостинга статей: собирает новости с заданных источников,
отдаёт их LLM (**Hermes** или **DeepSeek** — на выбор) для написания
SEO-статьи, отправляет на согласование (WEB-админка + Telegram), а после
подтверждения публикует — push в GitHub-репозиторий сайта (далее GitHub
сам заливает по FTP) + кросс-постинг в соцсети.

> Полная архитектура и план по фазам: [`ARCHITECTURE.md`](ARCHITECTURE.md).
> Исходная идея: [`idea.md`](idea.md).

## Статус

**Минимальный пайплайн работает:**
Источники (RSS) → «Собрать» → папки анализа → «Обработать» (LLM) → «Превью» → «Одобрить».

- Управление RSS-источниками в админке (вкладка «Источники»).
- Сбор: парсинг RSS, догрузка полной статьи если в ленте только кусок,
  извлечение чистого текста (`trafilatura`) и картинки **из источника**.
- Файловая структура: `data/analysis/<поток>/<партия>/<новость>/`
  (1 папка на поток, до 10 партий на поток с ротацией, в партии — новости).
- Настройки для LLM (вкладка «Настройки»): язык, символов на новость,
  «картинки строго из источника», доп. инструкции.
- Обработка через LLM (`hermes` / `deepseek` / …) с генерацией JSON-статьи.
- Вкладка «Превью» с карточками и кнопками «Одобрить» / «Отклонить».
- Публикация после одобрения — **заглушка** (см. ниже).

**Ещё не сделано (нужны вводные):** реальная публикация на сайт и в соцсети.
Требуется пример файла статьи на сайте (шаблон) + путь в репозитории, GitHub-токен,
токены X/Instagram. См. `ARCHITECTURE.md` §11. Также пока нет планировщика
(сбор запускается кнопкой), Telegram-согласования и аутентификации админки.

## Запуск (локально)

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # впишите LLM_PROVIDER / LLM_BASE_URL / LLM_KEY
uvicorn app.main:app --reload --port 8080
```

Откройте http://localhost:8080 — статус LLM и кнопка пробного вызова.

## Выбор провайдера

Меняется в `.env` без правки кода:

| LLM_PROVIDER | base_url по умолчанию | модель по умолчанию |
|--------------|-----------------------|---------------------|
| `deepseek`   | `https://api.deepseek.com/v1` | `deepseek-chat` |
| `hermes`     | — (задайте `LLM_BASE_URL`) | `hermes` |
| `openai`     | `https://api.openai.com/v1` | `gpt-4o-mini` |
| `claude`     | `https://api.anthropic.com/v1` | `claude-sonnet-4-6` |
| `local`      | `http://localhost:11434/v1` | `llama3.1` |

## Docker

```bash
docker compose up --build
```

Образ также собирается и публикуется в GHCR через GitHub Actions при push в `main`.
